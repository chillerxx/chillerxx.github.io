🌸 理解setState

Date:   2019-10-09 

调用流程图

详细方法解析

setState

ReactBaseClassses.js

enqueueSetState

enqueueUpdate

ReactUpdates.js

当前处于创建/更新组件阶段，就不会立刻更新组件，而是吧组件push进dirtyComponents
这也解释了 通常情况下setState是个异步过程，会累计后批量更新
不过在原生事件和setTimeout中是同步的

  • setState一个值多次 调用Object.assign()会覆盖之前的 只执行最后一个
  • setState多个不同值 在更新时会合并批量更新

batchingStrategy

ReactDefaultBatchingStrategy.js

第一次调用，isBatchingUpdates一定为false，会执行更新事务，callback为enqueueUpdate
此时isBatchingUpdates已经被置为true，会将组件push进dirtyComponents

🌟ReactDefaultBatchingStrategyTransaction是一个事务 有两个wrapper

接着前面callback执行完，现在该执行close。

flushBatchedUpdates

如果存在脏组件，执行ReactUpdatesFlushTransaction事务

🌟ReactUpdatesFlushTransaction 有两个wrapper

会再close阶段再次检查dirtyComponents长度有没有变化，如果存在新的dirtyComponents，需要再来一次flushBatchedUpdates

runBatchedUpdates

会将dirtyComponents中的组件按父 > 子排序，然后遍历组件调用performUpdateIfNecessary，同时将组件的回调压入回调队列，等待最后调用，意思是等待所有组件更新完，才回去调用组件回调。

performUpdateIfNecessary

updateComponent

updateComponent会依次执行组件的生命周期
componentWillReceiveProps => 将queue中缓存的state与缓存的state合并 =>
shouldComponentUpdate => componentWillUpdate => render => 得到的新的reactElement与之前的做diff，将不同diff类型（更新，删除，移动等）调用setInnerHTML等更新视图 => componentDidUpdate等。

transaction 事务

将要执行的method使用wrapper封装起来，再通过暴露的perform方法执行
一组initialize及close方法称为一个wrapper可以多个叠加
执行顺序：
所有wrapper的initialize() => perform() => 所有wrapper的close（）
