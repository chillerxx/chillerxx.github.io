🤓《深入理解ES6》读书笔记

Date:   2019-12-21 

ES6

Let var const
JavaScript有一个机制叫“预解析”，也可以叫“提升（Hoisting）机制”
• 函数提升优先于变量提升，
函数提升会把整个函数挪到作用域顶部（解决函数间互相调用的情况）
变量提升只会把声明挪到作用域顶部
• var 存在提升，我们能在声明之前使用。
let、const 因为暂时性死区的原因，不能在声明前使用
• var 在全局作用域下声明变量会导致变量挂载在 window 上，其他两者不会
• let 和 const 作用基本一致，但是后者声明的变量不能再次赋值
• 暂时性死区 let和const声明绑定在块级作用域内，形成了封闭作用域。

ES6中对块级作用域做了进一步强化
块级声明用于声明在指定块的作用域之外无法访问的变量
块中（字符 { 和 } 之间的区域）
比如 if 和 for 在ES6中也被定义成一个块级。

let声明 把变量的作用域限制在当前代码块中
声明不会被提升 所以可以防止变量泄露
禁止在同一个作用域下重复声明

循环中使用闭包解决 var 定义函数
With var i in every setTimeout callback means the same variable
With let i in every setTimeout callback means a different variable

原型 原型链
一个对象里有 proto 原型对象，在该对象中定义了很多函数
其中有 constructor 属性 里面包含 prototype 原型
原型链就是多个对象通过 proto 的方式连接了起来

• 函数的 prototype 是一个对象
• 对象的 proto 属性指向原型， proto 将对象和原型连接起来组成了原型链

Proxy 自定义对象中的操作

map, filter, reduce
reduce 可以将数组中的元素通过回调函数最终转换为一个值
接受两个参数，分别是回调函数和初始值
回调函数接受四个参数，分别为累计值、当前元素、当前索引、原数组
求和 或者 实现map
const arr = [1, 2, 3]
const mapArray = arr.map(value => value * 2)
const reduceArray = arr.reduce((acc, current) => {
acc.push(current * 2) return acc
}, [])
console.log(mapArray, reduceArray) // [2, 4, 6]

Generator 可以控制函数的执行
第一次 next 时，传参会被忽略，并且函数暂停在第一个 yield
第二次 next 时，传入的参数等于上一个 yield 的返回值，
如果你不传参，yield 永远返回 undefined
Generator 函数解决回调地狱问题
